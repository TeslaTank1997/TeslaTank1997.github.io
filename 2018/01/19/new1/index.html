ubuntu16.04使用hexo搭建github博客

Veröffentlicht am 2018-01-19

2018年1月19日，我终于把想要搭的博客搭好了……
搭建的路程还真是异常的曲折，首先想在windows下尝试，首先搜索到的教程是用jekyll作为模板来用，经过多重踩坑之后以放弃告终。然后可爱的室友告诉我hexo比较好用，于是转战hexo，结果在卡在了部署的那里，经过一番折腾后再次弃坑，一怒之下转向了前段时间折腾我好长时间的linux系统，终于……好了（心情激动中）。
讲道理中间还是踩了些坑，不过好在整体过程都没什么问题，现在趁热把过程记录下来。

安装hexo

hexo的官网还是足够地人性化的，把hexo的安装和配置都讲得比较清楚。
由于hexo是基于nodejs的，所以需要先行安装nodejs。
按照官网的方法，最好是用Node Version Manager来安装，过程如下：

$ curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.33.2/install.sh | bash
$ wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.33.2/install.sh | bash
$ nvm install stable

这样就下载到了nodejs的稳定版本，当然也可以在nodejs的官网上下载安装包然后去安装。
另外，我在下载的时候直接用了：

$ sudo apt-get install nodejs

这样也是可以用的，至少在当前的版本下是可以的。
接下来安装npm，用它来安装hexo。

$ sudo apt-get install npm
$ npm install -g hexo-cli

（上面的-g表示全局安装，如果没有-g的话就会被安装在当前执行命令行的路径下）
这样一来就安装好hexo了。

使用hexo搭建本地服务器

官网上很友好地详细地写出了详细的过程：
首先初始化：

$ hexo init

最右边那个是要放博客文件的文件夹名字，可以是还不存在的文件夹，会自动创建，也可以是已经创建好的，但是必须是空的。
接下来进入文件夹安装nodejs支持

$ cd
$ npm install

安装之后文件夹中就会出现一系列文档和文件夹，其中比较重要的是_config.yml，是网站的配置文件，有很多个性化选项都可以在里面进行配置，具体的可以在官网上看到。
接下来是重点：

$ hexo clean
$ hexo generate
$ hexo server

搭建github上自己的博客

这样的话就需要git了，这个安装起来简单暴力

$ sudo apt-get install git

为了能够创建博客，需要在github上创建一个仓库，创建的时候名字设置成为.github.io。
为了能够使本地用户能够直接访问仓库，需要创建一个sshkey：

mkdir ~/.ssh
cd ~/.ssh
ssh-keygen -t rsa

这样之后复制已经创建好的密钥，在github中的个人设置中添加密钥：

cat ~/.ssh/id_rsa.pub

这样就配置好了密钥，然后我们需要配置一下github用户：

git config –global user.name “xxx”
git config –global user.email “xxx@xxx.com”

然后进行测试，这里出现可以进入到github就好：

ssh -T git@github.com

这里好了之后，去复制一下仓库的地址，就可以放心地关掉github的网页了，下面是hexo的操作。
首先修改_config.yml文件，在文件最后修改：

#Deployment

##Docs: https://hexo.io/docs/deployment.html
deploy:
type: git
repo: git@github.com:xxx/xxx.github.io.git
branch: master

其中第二个是刚才复制的仓库的地址。
接下来是一系列操作，在以后的部署网站的操作中也是这样做：

hexo clean
hexo g
hexo d

其中g是generate，d是deploy，这两个也可以连着使用：

hexo d -g

这样一来，输入自己的仓库的名字，就可以看到自己的博客啦～虽然现在还是白板就是了……

发布文章

发布文章的话需要首先创建一个文件：

hexo new “xxx”

后面那个是文件名。这样会生成一个.md文件，文章支持markdown格式，还是比较好用的。
如果想要插入图片的话首先需要更改一下设置，在_config.yml中修改：

post_asset_folder: true

然后执行：

npm install hexo-asset-image –save

这样的话生成文章的同时会出现一个文件夹，里面存放需要用到的图片，然后在.md文件中按照markdown格式去引用就好。
在写好文章之后需要发布的话就使用刚才的指令进行部署就好。
同样还可以写草稿，这些不会被看见，也就是说可以当私密文档用，操作如下:

hexo new draft “xxx”

然后想要变成文章的话就：

hexo publish [layout]

更改主题

其实更改主题是很简单的一件事情，而我，却失败了很多次……最终使用了现在这个NexT主题。其实这个主题还是不错的，但是有同学在用，不太想重复……
想要更换主题的话，可以在官网的Theme一栏中找到好多好看的主题，然后点开预览在里面找到作者的github，就可以找到主题的相应文件啦。一般情况下的操作是，用git把github上的东西clone到自己的博客文件夹下的themes文件夹中，文件家的名字可以自己起，然后对_config.yml文件进行操作，更改theme:后面为刚才那个文件夹的名字，比如我用的是next，就是这样子的：

#Extensions

##Plugins: https://hexo.io/plugins/

##Themes: https://hexo.io/themes/
theme: next

然后就大功告成啦～
